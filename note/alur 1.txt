Step-by-step pengisian data Sekolah, Kejuruan, Kelas, dan Pertanyaan

1) Jalankan server backend
- Command: `npm run start:prod`
- Base URL: `http://localhost:3000`

2) Tambah Sekolah
- Endpoint: `POST /sekolah`
- Body contoh:
  {
    "nama_sekolah": "SMK Contoh"
  }
- Respons: `{ "id": <id_sekolah> }`

3) Tambah Kejuruan
- Mengacu skema: kejuruan terikat ke `angkatan` (lihat `note/dbangket.sql`)
- Endpoint: `POST /kejuruan`
- Body contoh:
  {
    "id_angkatan": 1,
    "nama_kejuruan": "Rekayasa Perangkat Lunak"
  }
- Respons: `{ "id": <id_kejuruan> }`
- Catatan: Pastikan `id_angkatan` sesuai data di tabel `angkatan` apabila sudah tersedia.

4) Tambah Kelas
- Kelas terikat ke `kejuruan`
- Endpoint: `POST /kelas`
- Body contoh:
  {
    "id_kejuruan": 1,
    "nama_kelas": "XII RPL 1"
  }
- Respons: `{ "id": <id_kelas> }`

5) Tambah Pertanyaan
- Pertanyaan bertipe `pilihan_ganda` dan memiliki 5 pilihan nilai: Very low, Low, Medium, High, Very high
- Pertanyaan dapat diikat ke `sekolah`, `kejuruan`, dan `kelas` tertentu
- Endpoint: `POST /pertanyaan`
- Body contoh:
  {
    "isi_pertanyaan": "Bagaimana tingkat kenyamanan?",
    "bobot_persentase": 10.0,
    "id_sekolah": 1,
    "id_kejuruan": 1,
    "id_kelas": 1
  }
- Respons: `{ "id": <id_pertanyaan> }`

6) Melihat Daftar Pertanyaan (opsional filter)
- Endpoint: `GET /pertanyaan`
- Query opsional: `id_sekolah`, `id_kejuruan`, `id_kelas`
- Contoh: `GET /pertanyaan?id_sekolah=1&id_kejuruan=1&id_kelas=1`

7) Update/Hapus
- Update Sekolah: `PATCH /sekolah/:id` body `{ "nama_sekolah": "SMK Update" }`
- Hapus Sekolah: `DELETE /sekolah/:id`
- Update Kejuruan: `PATCH /kejuruan/:id` body `{ "nama_kejuruan": "RPL Update" }`
- Hapus Kejuruan: `DELETE /kejuruan/:id`
- Update Kelas: `PATCH /kelas/:id` body `{ "nama_kelas": "XII RPL 2" }`
- Hapus Kelas: `DELETE /kelas/:id`
- Update Pertanyaan: `PATCH /pertanyaan/:id` body `{ "isi_pertanyaan": "...", "bobot_persentase": 12.5 }`
- Hapus Pertanyaan: `DELETE /pertanyaan/:id`

Catatan konsistensi skema
- Referensi skema lengkap ada di: `note/dbangket.sql`
- Pastikan ID yang digunakan (`id_angkatan`, `id_kejuruan`, `id_kelas`) merujuk data yang valid pada tabel terkait.
